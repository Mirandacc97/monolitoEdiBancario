## Passo a Passo Estruturado para o Projeto Final (Baseado no PDF 08-Projeto-Final-quinta.pdf)

### 1. **Definição do Projeto**
- Escolha e registre o tema: *Portal Web de Simulação de EDI Bancário*.
- Documente que o grupo é formado apenas por você.
- Elabore uma breve descrição do objetivo do projeto e das páginas planejadas (exemplo: Home, Envio de Remessa, Retorno Bancário, Conciliação, Contato).

### 2. **Criação do Repositório**
- Crie um repositório no Github com nome claro (ex: edi-bancario-web).
- Adicione um README com:
  - Tema do projeto
  - Nome do integrante (você)
  - Descrição do objetivo e das páginas
- Envie o link do repositório pelo Blackboard conforme exigido[1].

### 3. **Planejamento da Interface e Funcionalidades**
- Esboce o layout das páginas (pode ser em papel ou ferramenta digital).
- Defina as cores, fontes e crie um logo simples.
- Liste as funcionalidades mínimas:
  - Menu com pelo menos 4 links (páginas)
  - Formulário funcional (ex: envio de remessa ou contato)
  - Identidade visual própria (cores, logo)
  - CSS externo próprio (pode usar Bootstrap, mas personalize)
  - Código limpo e organizado[1]

### 4. **Desenvolvimento do Front-End**
- Crie os arquivos HTML para cada página.
- Implemente o menu de navegação.
- Desenvolva o formulário funcional.
- Aplique o CSS externo próprio.
- Garanta que o site esteja apresentável e responsivo[1].

### 5. **Documentação do Projeto**
- Crie um arquivo de documentação (ex: DOC ou Markdown no repositório) contendo:
  - O que é EDI bancário (resumo)
  - Como funciona a simulação no seu site
  - Decisões de design e funcionalidades
  - Descrição do desenvolvimento individual
- Tire prints das telas do sistema em diferentes etapas e anexe à documentação[1].

### 6. **Commits e Evolução**
- Faça commits frequentes no Github, descrevendo claramente cada etapa realizada.
- Mantenha o repositório atualizado até a data final[1].

### 7. **Testes e Revisão**
- Teste todas as páginas, links e formulários.
- Revise o código para garantir boas práticas de HTML, CSS e organização dos arquivos[1].

### 8. **Preparação para Apresentação**
- Prepare uma apresentação de até 5 minutos:
  - Explique o tema e objetivo do portal
  - Mostre as páginas e funcionalidades
  - Destaque o formulário e a identidade visual
  - Informe que o projeto foi realizado individualmente[1].

### 9. **Entrega Final**
- Confirme que o repositório está público e atualizado.
- Certifique-se de que a documentação está completa.
- Leve o site pronto para a apresentação presencial na data marcada (quinta-feira cedo)[1].

- [ ] Repositório criado e link enviado no Blackboard
- [ ] Estrutura do site com pelo menos 4 páginas
- [ ] Formulário funcional implementado
- [ ] Identidade visual própria criada
- [ ] CSS externo criado e aplicado
- [ ] Documentação e prints anexados
- [ ] Commits frequentes no Github
- [ ] Apresentação preparada para até 5 minutos
